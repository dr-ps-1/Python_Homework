# Lesson 10: Документация проекта

## Описание проекта
Проект содержит автоматические тесты для веб-приложений:
- **Калькулятор** (https://bonigarcia.dev/selenium-webdriver-java/slow-calculator.html)
- **Интернет-магазин Saucedemo** (https://www.saucedemo.com/)

## Описание базового синтаксиса записи и форматирования

Проект использует следующие технологии и синтаксис:
1. **Python**: Основной язык программирования для написания тестов.
2. **Selenium**: Библиотека для автоматизации взаимодействия с веб-браузером.
3. **Pytest**: Фреймворк для написания и запуска тестов.
4. **Allure**: Инструмент для генерации отчетов о выполнении тестов.

### Форматирование кода

- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Используются docstribgs для документирования методов и функций.
- Все шаги теста размечаются с помощью `@allure.step` (для методов PageObject) или `with allure.step` (для блоков в тестах) для улучшения читаемости отчетов.
- Тесты имеют декораторы Allure:
 - `@allure.title` - заголовок теста.
 - `@allure.description` - подробное описание.
 - `@allure.feature` - название функциональности.
 - `@allure.severity` - уровень критичности (BLOCKER, CRITICAL).

## Инструкция по запуску тестов для формирования отчета Allure
1. Установить необходимые зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Запустить тесты с генерацией отчета Allure:
   ```bash
   pytest --alluredir=./allure-results
   ```

   Эта команда запустит все тесты и сохранит результаты в директорию `./allure-results`.
   
   ## Инструкция по просмотру сформированного отчета Allure
   1. После завершения тестов, чтобы просмотреть отчет, необходимо выполнить команду:
   ```bash
   allure serve ./allure-results
   ```

   Эта команда запустит локальный сервер и откроет отчет в браузере.
   2. В отчете Allure представлено:
    * Статистика по тестам: количество пройденных/ упавших тестов.
    * Детали каждого теста:
     - * Название теста ("Тестирование калькулятора: 7 + 8 = 15", "Тестирование оформления заказа в интернет-магазине").
     - * Описание теста: что проверяет.
     - * Шаги теста:
     - Открытие страницы.
     - Установка задержки.
     - Нажатие кнопок.
     - Ожидание результата.
     - Проверка результата.
    * Результат: Успешное выполнение или ошибка с указанием ожидаемого и фактического результата.
